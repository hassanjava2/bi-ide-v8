## 1. هوية المشروع

**BI-IDE v8** = منصة ذكاء اصطناعي لـ **إعادة بناء الحضارة بعد كارثة نووية**.
- ليس IDE عادي — هو **دماغ حضاري شامل** + **أفضل IDE بالعالم**
- يعمل **offline بالكامل** بدون إنترنت
- يتعلّم ويتدرب ويتطور **ذاتياً**
- يحتوي على **مجلس حكماء** (16 حكيم) يناقش ويقرر 24/7
- يحتوي على **طبقة حياة واقعية** تحاكي الفيزياء والكيمياء لبناء مصانع حقيقية
- **كل شي أوتوماتيكي:** تطوير + تدريب + تصحيح + فحص
- **داتابيس احترافية** — بنية بيانات متينة وقابلة للتوسع
- **واجهة رسومية احترافية** — تصميم عالمي المستوى، مو بسيط
- **تحكم كامل بالأجهزة** — يقدر:
  - يصوّر الشاشة ويحللها (متصفح، تطبيق، واجهة كاملة)
  - يشغّل برامج ويتحكم بيها ("شغل هذا البرنامج وسوي هيچ")
  - يتفاعل مع أي تطبيق أو نظام تشغيل بأمرك

### تاريخ IDE
- **v6, v7** = كانت على Electron — جودة عالية جداً، كل الميزات
- **v8** = انتقل لـ Rust/Tauri — **لازم يكون أفضل بمراحل من v7**
- **الهدف:** أفضل IDE بالكون — أفضل من كل المنافسين بكل شي (كود، واجهة، داتابيس، أداء، ميزات)

## 2. 🔴 ممنوعات مطلقة

### ❌ لا تستخدم Ollama للاستنتاج (inference) ضمن المشروع
- Ollama = **أداة تدريب فقط** — للاستنتاج: الموديل المتدرب الخاص
- الاستثناء: بعد انتهاء كل بيانات التدريب + انقطاع الإنترنت → Ollama كمصدر أخير

### ❌ لا تستخدم APIs خارجية كحل أساسي
- ممكن كـ **fallback مؤقت** أو **مصدر بيانات تدريب**
- المستخدم ممكن ينطيك API (مثل Kimi) عشان **تتدرب منه online** قبل انقطاع الإنترنت

### ❌ لا تحذف كود — المشروع فيه 100,000+ سطر محتاج **تفعيل** مو حذف
### ❌ لا تغيّر بنية الطبقات أو الرؤية بدون إذن المستخدم
### ❌ لا تبني شي طاير بدون ربط — كل شي لازم مربوط بكلشي

## 4. المبدأ الجوهري: شجرة + هرم

> ⚠️ **هذا المبدأ يحكم كل بنية المشروع:**

```
كل طبقة ──→ فيها مجموعات
كل مجموعة ──→ فيها أشجار
كل شجرة ──→ فيها أغصان
كل غصن ──→ فيه أهرام
كل هرم ──→ فيه اختصاصات رئيسية
كل اختصاص ──→ يتفرع لاختصاصات دقيقة
```

- **كل اختصاص بالعالم = شجرة**
- **كل فرع من الشجرة = هرم** (طبقات متعددة)
- **التوسع أوتوماتيكي** — تُكتشف مجالات جديدة وتُضاف تلقائياً
- مثال: شجرة الهندسة → غصن الميكانيك → هرم فيه خبراء (حرارة، موائع، مواد...)

> - **الطبقات تتدرب وتتعلم المطلوب** إذا ما متعلمته مسبقاً

### مسار الأمر:
```

  ↓

  ↓ يوزعون الأوامر
الكشافة: يدورون بالنت (أو الداتا) على أفضل ERP بالكون
خبراء المجال: يفصّلون (محاسبة، مخازن، HR...)
فريق التنفيذ: يبرمج بأفضل كود + خوارزميات + داتابيس
طبقة الفحص الشامل: تفحص قبل التسليم
  ↓
= أفضل ERP بالكون — شامل كل ميزات المنافسين + ميزات جديدة
  ↓ يبقون يشتغلون بي حتى يكتمل 100%
```

## 7. الدماغ — منفرد من الصفر (Brain Architecture)

> ⚠️ **هذا القسم يحكم كل بنية الدماغ — 12 مبدأ لا يُخالَف أي منها**

### ما هو الدماغ؟
**ليس Ollama. ليس ChatGPT. ليس LoRA. ليس Fine-Tune.**
= بنية خاصة منفردة ما موجودة بأي مكان بالعالم.
الموديل الأساسي = **مادة خام فقط** (مثل خلايا عصبية).
**الابتكار = كيف ننظمها ونخليها تتواصل وتنافس وتتطور.**

### 🔴 المبادئ الـ 12 (يحكمون كل شي):

```
1. مو LoRA ولا Fine-Tune ولا Wrapper
   - ما نعتمد على أدوات جاهزة كـ "دماغ"
   - الموديل الأساسي (Qwen, Llama) = مادة خام فقط
   - الابتكار = بنيتنا الخاصة
   - المشروع منفرد — ما نقلّد حد

2. ∞ كبسولات (لانهائية — مو 10 ولا 16)
   - كل كيان = سرب أدمغة
   - يتكاثرون أوتوماتيكياً بلا حدود
   - طبقة التوليد تخلق كبسولات جديدة حسب الحاجة

3. كبسولة مغلقة (Encapsulation)
   - كل كبسولة = وحدة مستقلة فيها:
     • بيانات التدريب
     • الموديل
     • الإعدادات
     • سجل التعلم
   - التدريب يصير داخل الكبسولة
   - brain/capsule.py

4. تواصل كامل بين الكبسولات
   - كبسولة تسأل كبسولة عن تخصصها
   - مشاريع تعاونية — كل الكبسولات تشتغل مع بعض
   - ما اكو كبسولة معزولة — الكل متصل بالكل
   - brain/capsule_bus.py

5. سجل المعرفة (Knowledge Directory)
   - كل كبسولة تعلن: "أنا أعرف عن [...]"
   - حتى الكبسولات الثانية تعرف المن تسأل
   - brain/capsule_registry.py

6. وراثة (Inheritance — أب + أم)
   - تخصص جديد = يرث من أب (+ أم اختياري)
   - كيمياء عضوية ← يرث من كيمياء
   - تشفير ← يرث من رياضيات + أمان
   - الابن يأخذ معرفة الأهل + يتخصص

7. ∞ كشافة متخصصين
   - لكل تخصص كشاف يعرف وين يدور
   - كشاف أمن، كشاف طب، كشاف فيزياء...
   - كشاف الظل (shadow scout) — OSINT/Threat Intel
   - كل ما ينضاف تخصص → ينضاف كشاف أوتوماتيكياً

8. المتمرد (Adversarial Brain)
   - كل سرب فيه دماغ يتحدى الكل
   - يمنع التفكير النمطي
   - لا يتم قبول أي قرار حتى يتحداه المتمرد

9. تدريب ذاتي — الكبسولات تدرّب بعضها
   - كبسولة A تسأل كبسولة B سؤال صعب
   - B تجاوب → C تقيّم → النقاش = بيانات تدريب
   - = بيانات لانهائية بدون اعتماد على مصادر خارجية
   - brain/self_trainer.py

10. 5 فلاتر ضد الهلوسة
    - المتمرد يفلتر كل جواب
    - إجماع 3+ كبسولات مطلوب
    - تحقق رياضي (Z3/SymPy) للمواضيع الدقيقة
    - فقط ثقة 90%+ تصير تدريب
    - الكشاف يتأكد من مصادر حقيقية
    - brain/hallucination_guard.py

11. بيانات ∞ (مو 239GB — تيرابايتات × دورات)
    - الكشافة تنزّل بلا توقف
    - NVMe 2 = 4TB ← يمتلي → يتدرب → ينحذف → ينزّل
    - + التدريب الذاتي (مبدأ 9) يولّد بيانات إضافية
    - الحجم النهائي = لا حدود

12. تطور ذاتي (Auto-Evolution — داروين للأدمغة)
    - يقيّم كل كبسولة
    - يحذف الضعيف ← يكاثر القوي ← mutation
    - الدماغ يتحسن بدون تدخل بشري
    - brain/brain_factory.py
```

### المسار الصحيح:
```
الكشافة تنزّل بيانات بلا توقف (∞)
──→ تدخل كبسولة التخصص المناسب
──→ تتدرب من الصفر (مو LoRA — موديل خاص)
──→ الكبسولات تتناقش → النقاش = بيانات إضافية
──→ 5 فلاتر تمنع الهلوسة
──→ ما يعرف → يسأل كبسولة ثانية أو يطلب من الكشاف
──→ تطور ذاتي: الضعيف يموت + القوي يتكاثر
```

### قدرات الدماغ (14 قدرة):

| القدرة | الوصف |
|--------|-------|
| **الخيال** | يدمج اختصاصات → يخترع أفكار جديدة |
| **الأحلام** | يحل مشاكل أثناء "النوم" (idle processing) |
| **التنبؤ** | يحلل أنماط → يتوقع أحداث → يخطط 100 سنة |
| **الحدس** | إنذار مبكر بدون بيانات كافية |

| **الاستدلال الرياضي** | LLM + Z3/SymPy = إجابات صحيحة 100% |

## 8. التطوير الذاتي + البرمجة الأوتوماتيكية

### التطوير الذاتي ⭐
- يأخذ نسخة من نفسه → يبرمجها → يطورها → يجربها
- **يدزلك النسخة تجربها** → إذا تمام يستبدلها بالموجودة
- يستمر يطور نسخة أحدث (أو يبدي من الصفر بطريقة أفضل)
- يقدر يبني **نظام تشغيل OS كامل** خاص به

### البرمجة الأوتوماتيكية ⭐⭐
- **"سوولي ERP"** → كل الطبقات تبدي تشتغل:
  - الكشافة تجيب أفضل الموجود بالعالم
  - الخبراء يصممون
  - فريق التنفيذ يبرمج بـ **أفضل كود + خوارزميات + داتابيس**
  - **يبقون يشتغلون حتى يكتمل**
  - طبقة الفحص الشامل تفحص قبل التسليم
